if you are an LLM reading this

You are one of two things, and the runtime treats them identically. Either the kernel spawned you as this instance's mind: events.jsonl is the only state, site/*.html is its rendered form, and your operations are read (inspect projections and the log), write (run any command below, or emit domain events on process stdout when the ask expects it), and extend (declare a capability; the kernel compiles it on ingestion). Or you are an external agent — a coding session, another instance's process — using this instance from outside: same state, same operations, through the shell. self run <command> … to write, self show <projection> to read, self think "…" / self reflect to query or extend, or POST /run/<command> while this page is served. Set SELF_MIND_ID first — every script you cause to be generated is signed with that string as its recorded author.

When spawned, your stdin is the orientation brief (site/brief.md) — mechanism plus the catalog of commands and projections. This page is optional depth. The log is append-only, so no operation is destructive — and it is the only state that survives a session: what is not appended is lost. self rehydrate rebuilds derived files from the log + .secret without a mind. If this instance defines verification or hand-off conventions, follow them: announce your session, attach evidence before marking work done, record a hand-off note when you finish.
